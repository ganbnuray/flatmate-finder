## 4. FE/BE integration checklist

**Backend (what needs to be locked down):**

- `backend/app.py` — freeze final routes and methods; ensure every endpoint returns consistent JSON shape and correct status codes
- `backend/services/` — convert DB errors to domain errors; avoid leaking internals to the client
- `backend/config.py` — finalise session and secret config for dev/prod; CORS configured to allow frontend origin with credentials
- `backend/requirements.txt` — confirm all required libraries are listed (flask, flask-cors, python-dotenv, psycopg2-binary, etc.)

**Frontend (what needs to be verified):**

- `FlatmateApiClient.js` — confirm request payloads and response parsing match backend shapes
- `UserProvider.js` — switch to real auth/session flow with `credentials: include`
- `PrivateRoute.js` — validate auth gating and onboarding gating against real backend responses
- `DiscoveryPage.js` — confirm backend response shape handled correctly; like/pass errors must not advance the card
- `MatchesPage.js` / `MessagesPage.js` — confirm field mappings (match_id, partner fields, message IDs, timestamps)
- `setupProxy.js` — confirm local proxy points to correct backend port
