**Date:** April 10, 2026

**Note:** Assignment due in 11 days. Agreed to increase check-in frequency.

## MVP 2 priorities

1. **Fix messaging bugs** — pin input bar to bottom, enable scroll in thread, add auto-refresh or refresh button, add unread message indicator on matches list
2. **Quick profile fixes** — change currency £ to $, improve text contrast on cards, show gender on discovery cards, link match name/avatar to full profile from Matches and Messages pages
3. **Add email verification on signup** — a user registered with a fake email successfully; block account creation without a verified email
4. **Add filters to discovery** — the #1 requested feature across all user interviews, described as an adoption blocker; minimum set: budget range, gender, move-in date, smoking, pets
5. **Add "Looking for a place + flatmate together" housing status** — several users are in this situation and the current onboarding options do not cover it
6. **Add neighbourhood/area field to profiles** — "city" is too vague; users want to see and filter by area
7. **Harden match and messaging backend** — no duplicate matches, enforce that only match participants can read/send messages, enforce is_complete gating on discovery, apply block relationships across discovery, matches, and messaging
8. **Stage 3 interviews** — not urgent for now; once the full end-to-end flow works, run the next round focused on trust/safety and retention to scope V3
